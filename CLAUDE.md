# 📄 CLAUDE.md: PolicyIQ Architecture & API Blueprint

## 🏛️ Project Vision
**PolicyIQ** is a multi-agent, AI-powered policy stress-testing engine simulating 50 "Digital Malaysians." The system translates raw policy text into an 8-Knob state matrix, dynamically generates targeted economic sub-layers via RAG (Vertex AI + DOSM data), and simulates real-world sentiment and behavioral shifts over multiple time steps ("Ticks").

---

## 🛠️ Team Work Streams & Responsibilities

### 🧠 Stream 1: Team AI (Simulation, Agents & Knobs)
**Core Focus:** The Python-based physics engine, environment math, and LLM intelligence.
* **The Global State Engine:** Manages time progression (Ticks) and the mathematical state of the 8 Universal Knobs.
* **Dynamic Decomposition:** Uses Gemini to translate policy text into exactly 3 to 5 highly targeted sub-layers (enforcing the `target_demographic` and `impact_multiplier` rules).
* **Agent DNA & Economic Entities:** Maintains the JSON profiles for the 50 unique demographic citizens. Each agent is now a full **Economic Entity** with income, savings, debt, dependents, digital readiness, and subsidy eligibility — not just a demographic label.
* **The Observation Generator:** Retrieves RAG data from Vertex AI and constructs the localized, math-adjusted prompt for each agent per tick. The prompt explicitly instructs Gemini to use each agent's economic signals when computing `sentiment_score` and `internal_monologue`.
* **Genkit Orchestration:** Executes the agents in parallel using Firebase Genkit, ensuring strict JSON compliance.

### ⚙️ Stream 2: Team Backend (Infrastructure & API Gateway)
**Core Focus:** Data routing, cloud infrastructure, and endpoint exposure.
* **Cloud Setup:** Create the Google Cloud Project, enable billing, and manually upload cleaned DOSM datasets to Vertex AI Search.
* **API Gateway:** Build the REST API (using Django or FastAPI) that receives requests from the Frontend and routes them to the AI Engine.
* **The Gatekeeper:** Expose a separate `POST /validate-policy` endpoint for initial prompt validation.
* **Deployment:** Containerize the API and Genkit workflows and deploy them to Google Cloud Run.

### 📊 Stream 3: Team Frontend (The Decision Dashboard)
**Core Focus:** Data visualization, state representation, and user experience.
* **The Gatekeeper UI:** Build the input box that handles validation states and offers "refined policy" options if the user input is vague.
* **Control Panel:** Build the UI to manually override the 8 Knobs before simulation.
* **Macro Analytics:** Render Sentiment Heatmaps, Inequality Deltas, and Sankey diagrams for behavioral shifts.
* **Micro Insights (Anomaly Hunter):** Build the UI to spotlight individual citizens, displaying their strict JSON outputs and internal monologues.
